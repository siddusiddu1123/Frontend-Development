JavaScript
Variables & Data Types
Variable

The let Keyword

Use let when you expect the variable to be reassigned. It is block-scoped, meaning it only exists within the { } where it was created.

let age = 20;
age = 25; // Reassignment works!

console.log(age);
Variable

The const Keyword

Short for "constant". Use this by default for values that should not change. It prevents accidental bugs by locking the value.

const pi = 3.14;
// pi = 3.15; // This would throw an error!

console.log(pi);
Legacy

The var Keyword

The old way of declaring variables. It has function scope and can be "hoisted," which often leads to confusing bugs. Avoid using it in modern JS.

var name = "John";
var name = "David"; // Redeclaration is allowed (bad!)

console.log(name);
Data Type

Strings

Used to represent sequence of characters (text). You can use single quotes, double quotes, or backticks. 

let user = "Siddaih";
let greeting = `Hello ${user}`; // Template literal

console.log(greeting);
Data Type

Numbers

JavaScript has only one type of number. It can be written with or without decimals.

let age = 22;
let price = 99.99;

console.log(typeof age); // "number"
Data Type

Booleans

Represent logical entities and can have only two values: true or false. Great for conditional logic.

let isStudent = true;
let hasFinished = false;

if(isStudent) {
  console.log("Welcome to the class!");
}
Structure

Arrays

Arrays are used to store multiple values in a single variable, accessed by an index number (starting from 0).

let fruits = ["Apple", "Mango", "Orange"];

console.log(fruits.length); // 3
console.log(fruits); // "Mango"
